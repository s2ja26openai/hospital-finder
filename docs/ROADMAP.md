### 2.4 트레이드오프 기록

#### 프론트엔드: Jinja2 + Vanilla JS vs React/Vue SPA

| 옵션 | 장점 | 단점 |
|------|------|------|
| **Jinja2 + Vanilla JS (선택)** | 단일 서버 배포, 빌드 불필요, 즉시 시작 가능 | 복잡한 상태 관리 시 JS 코드 증가 |
| React/Vue SPA | 풍부한 상태 관리, 컴포넌트 재사용 | 빌드 파이프라인 필요, 단일 서버 요건과 충돌 |

> **결정 근거**: PRD에서 단일 서버 통합 운용을 명시. 현재 PRD 범위의 상태 관리 복잡도는 Vanilla JS로 충분히 처리 가능.

#### 지도 API: 카카오맵 vs 네이버 지도

| 옵션 | 장점 | 단점 |
|------|------|------|
| **카카오맵 (선택)** | 도보 경로 API 명확 제공, JS SDK 문서 우수 | 리뷰 데이터는 네이버에서 별도 수집 필요 |
| 네이버 지도 | 리뷰 + 지도 단일 플랫폼 | 도보 경로 API 접근 제한적 |

> **결정 근거**: 도보 경로(F-05-A)가 핵심 기능이므로 API가 명확한 카카오맵 선택. 리뷰는 네이버에서 별도 크롤링.

#### 리뷰 수집: 크롤링 vs 공식 API

> 네이버 지도 리뷰 공식 API가 미제공되므로 크롤링이 유일한 선택지. 네이버 정책 변경 시 크롤러 유지보수 비용 발생 가능.
